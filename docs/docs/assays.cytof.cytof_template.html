CIDC Documentation

CYTOF assay template

Description: Cytometry by Time-Of-Flight (CyTOF) aka mass-cytometry assay submission.

Property Description Data Type
assay run id User defined unique identifier for this assay run. Not necessarily unique across batches. < string >
batch id CyTOF batch identification number. Unique to every batch. < string >
protocol identifier Trial identifier used by lead organization, ie. Center for Experimental Therapeutics Program (CTEP) ID or Industry Sponsored ID. This is usually a short identifier. Example: E4412. < string >
assay creator Indicates what site is filling out the assay. < string >
  • "DFCI"
  • "Mount Sinai"
  • "Stanford"
  • "MD Anderson"
  • "Adaptive"
  • "Broad"
instrument Name of CyTOF instrument on which experiment was conducted. < string >
injector Name of Injector component used as part of the CyTOF software < string >
date of acquisition Date of CyTOF batch acquisition. < string >
acquisition buffer Cell staining buffer used for antibody and cell dilution. < string >
bead removal Boolean to indicate whether beads have been removed after normalization process < boolean >
normalization method Process used for normalization prior to analysis, if any. < string >
  • "Fluidigm"
  • "Normalizer v0.3"
  • "Other"
debarcoding protocol The strategy/kit used to barcode CyTOF samples, if any < string >
harware version Version no. of instrument hardware used. < string >
concatenated spike in fcs filename Processed FCS file for the spike-in. fcs
concatenated raw fcs filename(s) Path to a file on a user's computer. local file#file path

Run info
Property Description Data Type
cimac id Specimen identifier assigned by the CIMAC-CIDC Network. Formatted as CTTTPPPSS.AA for trial code TTT, participant PPP, sample SS, and aliquot AA. < string > as regex
^C[A-Z0-9]{3}[A-Z0-9]{3}[A-Z0-9]{2}.[0-9]{2}$
Preprocessing
Property Description Data Type
processed fcs filename Final FCS files after all processing. Spike-in's should be removed. fcs
concatenation version Concatenation of FCS files into a single FCS version number. < string >
normalization version Version of normalization for CyTOF assay batch. < string >
preprocessing_notes Any notes pertaining to preprocessing of CyTOF data. < string >

Samples
Property Description Data Type
target Antibody type collected for this study. < string >
clone Resulting type of antibody clone from primary antibody sample. < string >
antibody company Company from which antibody sample was derived. < string >
catalog number Antibody registry catalog number assigned to antibody sample. < string >
lot number Identification number assigned to the particular quantity or lot of material from manufacturer. < string >
isotope Antibody isotype used to help differentiate antibody signals. < string >
dilution Concentration ratio of dilution buffer for primary antibody. < string >
stain_type Type of staining method used for antibody. < string >
  • "Surface Stain"
  • "Intracellular"
usage How this antibody should be used in automatic analysis < string >
  • "Ignored"
  • "Used"
  • "Barcoding"
  • "Analysis Only"
